JEmacs and the gnu.emacs Package

The gnu.expr provides various building blocks for building an Emacs-like text editor. It also provides a number of Scheme procedures that have the same name and similar functionality as the Emacs Lisp functions.

It uses the Swing tool-kit.

Plan

The plan is to use this together with the (unfinished) gnu.elisp package to provide a fully-functional re-implementation of Emacs Lisp which compiles to Java bytecodes. I'm hoping volunteers will help me make "JEmacs" into something useful. I have fairly clear ideas about much of what need to be done. Contact me if you are interested.

Status

Very unfinished proof of concerpt.

Usage

Load emacs.scm into a running Kawa. Then evaluate: 
(emacs)
That brings up a Emacs window.

You can do (split-window) to create new (sub-)windows in an existing frame; type (make-frame) to create a new frame (to-level window), or do (get-buffer-create NAME) to create a new buffer. Note also how ctrl/B and ctrl/F have been bound to the correct functions.
